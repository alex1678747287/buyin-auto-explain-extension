# 百应直播讲解循环器

Manifest V3 Chrome 扩展，用于抖音电商百应直播商品控制页，按固定间隔自动点击商品“讲解”按钮。

## 功能

- 默认每 15 秒执行一次，可在扩展弹窗中调整。
- 可指定商品序号，例如填 `2` 表示只循环讲解列表里序号为 2 的商品；留空表示自动选择第一个可用“讲解”。
- 如果指定商品已经处于“讲解中”，会先点击“取消讲解”，等待默认 2 秒后重新点击同一商品卡片里的“讲解”。
- 设置只保存在本机 Chrome `storage.local`，不会上传账号、Cookie 或页面数据。

## 安装

1. 下载或克隆本仓库。
2. 打开 Chrome 的 `chrome://extensions/`。
3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本仓库根目录，也就是包含 `manifest.json` 的目录。

## 使用

1. 打开百应直播商品控制页：

   ```text
   https://buyin.jinritemai.com/dashboard/live/control
   ```

2. 点击浏览器右上角扩展图标。
3. 设置参数：
   - `点击间隔（秒）`：两次自动操作之间的间隔，默认 `15`。
   - `商品序号（留空自动选择）`：填写左侧商品列表序号，例如 `1`、`2`、`3`；留空则自动选择第一个可用商品。
   - `取消后重讲等待（秒）`：检测到已在讲解时，先取消再重新讲解前的等待时间，默认 `2`。
4. 点击“启动”。
5. 需要停止时点击“暂停”。

## 行为说明

- 未填写商品序号时：插件会优先处理当前“讲解中”的商品；没有讲解中商品时，点击页面上第一个可见、可用的“讲解”按钮。
- 填写商品序号时：插件只会操作该序号对应商品。如果该商品正在讲解，就取消后重新讲解；如果未讲解，就直接点击“讲解”。
- 商品排序变化后，序号对应的商品也会随页面变化。
- 如果页面已经打开后才安装插件，弹窗会尝试自动注入脚本；仍失败时刷新百应页面再启动。

## 权限

`manifest.json` 使用以下权限：

- `storage`：保存本机配置。
- `activeTab` 和 `scripting`：在当前百应控制页注入脚本，兼容页面先打开后再加载扩展的场景。
- `https://buyin.jinritemai.com/*`：仅匹配抖音电商百应域名。

## 常见问题

### 找不到指定序号的商品

确认当前页面商品列表已经加载完成，并且商品序号仍然存在。插件会先尝试识别卡片内文本序号，识别不到时按商品卡片顺序取第 N 个。

### 找不到“讲解”按钮

百应页面结构或按钮文案可能改版了。请刷新页面后重试；如果仍失败，需要更新 `src/automation-core.js` 中的按钮识别规则。

### 能否后台运行

不能。插件运行依赖当前打开的百应控制页 DOM，Chrome 关闭标签页或页面未加载时不会继续点击。

## 注意事项

- 使用前请确认当前直播间和平台规则允许此类自动化操作。
- 不要在不熟悉页面状态时长时间无人值守运行。
- 本仓库不包含也不需要任何百应账号、Cookie、API key 或直播间私密信息。
